Guida interattiva alle impostazioni di Hesk

 

» Impostazioni generali

Nome Sito Web

Si tratta di un breve titolo che identifica il vostro sito web. Per esempio: "Il mio sito web". Viene usato per identificare il link che viene visualizzato nell'angolo alto a sinistra della pagina web dei clienti per consentire loro di poter tornare facilmente alla Home Page del vostro sito web.

URL Sito Web

Si tratta della URL (= Uniform Resource Locator) del vostro sito web. Per esempio "http://www.example.com". Accertarsi che si tratti di una URL valida (= esistente). Deve sempre contenere all'inizio la stringa http:// o https://. Viene usata come URL per il link che visualizzato nell'angolo in alto a sinistra della pagina web dei clienti per consentire loro di poter tornare facilmente alla Home Page del vostro sito web.

Indirizzo email del Webmaster

Si tratta dell'indirizzo di email predefinito del webmaster. Accertarsi che si tratti di un indirizzo di email valido. Può essere lo stesso, ma può essere anche diverso rispetto a quello del servizio di Helpdesk.

Indirizzo email dell'Helpdesk

Si tratta dell'"indirizzo di email" del servizio di Helpdesk. Questo è l'indirizzo di email che viene visualizzato come campo Da: nell'intestazione di tutti i messaggi di posta elettronica inviati da Hesk. Di solito si tratta di un indirizzo di email che non può ricevere risposte. Accertarsi che si tratti di un indirizzo di email valido. Può trattarsi dello stesso indirizzo di email usato dal supporto, ma è normalmente opportuno che sia differente.

Nome reale per l'email dell'Helpdesk

Questo è il nome che viene visualizzato come mittente del messaggio di email nel campo Da: dell'intestazione di tutti i messaggi di posta elettronica inviati da Hesk. Se non viene configurato alcun nome, nel campo Da: dell'intestazione verrà visualizzato solo l'indirizzo di email (cfr. sopra).

» Impostazioni Lingua

Lingua predefinita

Qualora si abbiano più lingue installate in Hesk, questa rappresenta la lingua predefinita. Tutte le lingue correttamente installate vengono visualizzate in una lista di selezione a discesa. Si veda il file readme.html per le istruzioni riguardanti l'installazione di più lingue. Il link a: "Cartella di test per la lingua" consente di ricevere informazioni sul come risolvere gli eventuali problemi di installazione di lingue non funzionanti.

Se si abilitano più lingue, è possibile sceglierne una come preferita nella pagina del proprio "Profilo".

Lingue multiple

L'abilitazione di questa impostazione consente ai clienti e agli operatori dello staff di scegliere la loro lingua preferita nell'ambito delle lingue installate. Non conviene abilitare questa funzionalità a meno che non si intenda fornire un effettivo supporto in tutte le lingue installate.

» Impostazioni Database

Server del Database

Nome del server che ospita il database. Nella maggior parte dei casi il nome "localhost" va normalmente bene. NON conviene modificarlo a meno che non si sappia qual è il server MySQL. Chiedere al proprio servizio di hosting se non si è sicuri o se si hanno problemi di conmnessione al database.

Nome del Database

Nome del vostro database MySQL. Per usare Hesk è infatti necessario crearne uno.

Nome utente per il Database

Nome dell'utente del database che si intende utilizzare per connettersi al proprio database MySQL. È ovviamente necessario creare nel pannello di controllo dell'hosting almeno un utente per connettersi al database di Hesk. Eventualmente si possono chiedere informazioni al proprio servizio di hosting.

Password per il Database

Password utilizzata dall'utente che accede e gestisce il database Hesk di MySQL (cfr. sopra). Eventualmente si possono chiedere informazioni al proprio servizio di hosting.

Prefisso tabelle

È il prefisso che viene aggiunto al nome di tutte le tabelle facenti parte del database Hesk. Se si accetta il valore predefinito, cioè hesk_, la tabella "users" del database verrà denominata "hesk_users". La presenza del prefisso è utile qualora si vogliano far girare istanze multiple di Hesk usando un solo database MySQL. In questo caso infatti è sufficiente fornire a ciascuna installazione un diverso prefisso. Per esempio: hesk1_, hesk2_, hesk3_, ecc...
