Guida interattiva alle impostazioni di Hesk

 

Invia email usando

Questa opzione consente di decidere se Hesk deve inviare i messaggi di email per mezzo della funzione PHP mail() o mediante la configurazione manuale di un server SMTP.

La funzione PHP mail() dovrebbe andar bene nella maggior parte dei casi; se non dovesse funzionare si può provare ad usare al suo posto un server SMTP.

Variabili SMTP disponibili:

Server SMTP: è il nome del server SMTP. In genere è localhost o qualcosa come mail.example.com
Porta SMTP: è la porta usata dal server SMTP. Generalmente si tratta della porta 25 per le connessioni non sicure, 587 per le connessioni TLS e 465 per le connessioni SSL
Timeout SMTP: timeout (in secondi) del server SMTP
Protocollo SSL : abilitare questa opzione se il server SMTP richiede il protocollo di sicurezza SSL
Protocollo TLS: abilitare questa opzione se il server SMTP richiede il protocollo di sicurezza TLS
Nome utente su SMTP: Il vostro nome utente sul server SMTP (se usato)
Password su SMTP: La vostra password sul server SMTP (se usata)

POP3 Fetching

Questa opzione abilita/disabilita la funzionalità di "POP3 Fetching", creando un ticket a partire da una email utilizzando un account di email gestito tramite una connessione di tipo POP3. Per le istruzioni riguardanti le impostazioni dell "POP3 Fetching" si prega di riferirsi al file readme.html.

Le variabili utilizzabili sono:

Timeout processo: tempo (in minuti) che si deve aspettare perché un processo di "POP3 Fetching" venga completato. Non inizia alcun nuovo processo prima che il valore di timeout sia stato raggiunto. Impostando questo valore a 0 si disabilita questo parametro e si consente sempre la partenza di un nuovo processo di "POP3 Fetching".

Server POP3: nome del server POP3, in genere mail.example.com
Porta POP3: porta del server POP3, in genere 110 per le connessioni non sicure e 995 per le connessioni sicure.
Protocollo TLS: se il server POP3 richiede il protocollo TLS, abilitare questa opzione.
Conserva una copia: se questa funzione è abilitata, Hesk NON cancellerà i messaggi di posta dal server POP3.
Nome utente POP3: il nome utente con cui si accede al server POP3.
Password POP3: la password con cui si accede al server POP3.

IMAP Fetching

Questa opzione abilita/disabilita la funzionalità di "IMAP Fetching", creando un ticket a partire da una email utilizzando un account di email gestito per mezzo di una connessione IMAP. Per le istruzioni riguardanti le impostazioni dell' "IMAP Fetching" si prega di riferirsi al file readme.html.

Le variabili utilizzabili sono:

Timeout processo: tempo (in minuti) che si deve aspettare perché un processo di "IMAP Fetching" venga completato. Non inizia alcun nuovo processo prima che il valore di timeout sia stato raggiunto. Impostando questo valore a 0 si disabilita questo parametro e si consente sempre la partenza di un nuovo processo di "IMAP Fetching".

Server IMAP: nome del server IMAP, in genere qualcosa come mail.example.com
Porta IMAP: porta del server IMAP, in genere 143 per le connessioni non sicure e 993 per le connessioni sicure
Crittografia: configura il tipo di crittografia IMAP richiesto (SSL/TLS)
Conserva una copia: se questa funzione è abilitata, Hesk NON cancellerà i messaggi di posta dal server IMAP
Nome utente IMAP: il nome utente con cui si accede al server IMAP
Password IMAP: la password con cui si accede al server IMAP

Email Loop

Queste impostazioni aiutano a rilevare eventuali loop (cicli continui inarrrestabili) riguardanti messaggi di posta generati a seguito di una errata configurazione dei sistemi auto-risponditori o dell'impossibilità di spedire i messaggi di posta

Qualsiasi numero di email superiore a quello definito in Max Hits (numero massimo) provenienti da un unico indirizzo di email e che arrivino entro un determinato Timeframe (lasso di tempo) espresso in secondi sarà ignorato dall'"email piping" per evitare fenomeni di spam e/o di email bombing

Impostare Max Hits a 0 per disabilitare il rilevamento del loop.

Attenzione: la funzionalità di rilevamento di eventuali email loop viene attivata solo in presenza di Email Piping o di "POP3 Fetching" attivo.

Trova refusi nell'indirizzo di email

Questa funzione cerca di rilevare eventuali refusi nella digitazione del dominio di un indirizzo di posta elettronica e suggerisce la versione corretta.

Per esempio se un cliente inserisce nome.cliente@gmial.com, Hesk rileverà l'errore e suggerirà di modificare il dominio in nome.cliente@gmail.com.

Hesk effettua un confronto tra il dominio dell'indirizzo di email digitato e l'elenco dei domini (corretti) presente nel suo elenco di Fornitori indirizzi di email.

 

Fornitori indirizzi di email

Si tratta dell'elenco di fornitori di indirizzi di email piuttosto noti che i vostri clienti potrebbero usare quando inviano un ticket. Hesk utilizza tale elenco per cercare di rilevare eventuali errori di digitazione durante la composizione dell'indirizzo di email.

Inserire un dominio per riga, SENZA il segno @ (at, chiocciola).

Si può inserire qualsiasi numero di domini, ma conviene non eccedere: conviene inserire solo i nomi dei domini di uso più frequente.

Manda una notifica al cliente quando:

Scegliere quando Hesk deve inviare una email di notifica ai clienti.

  • Viene aperto un nuovo ticket

    Nota: se il ticket viene inviato tramite email piping o POP3 fetching, Hesk può evitare di inviare al cliente la notifica della ricezione del nuovo ticket se l'oggetto del messaggio di email del ticket contiene una delle etichette presenti nella casella di testo (un'etichetta per riga).
    È possibile usare questo metodo per prevenire l'invio di una risposta automatica alle email, il che viene spesso interpretato come SPAM dal filtro anti-SPAM dei server di posta.
     
  • Un ticket dè stato contrassegnato come "Risolto"

 

NOTA: Le preferenze relative alle notifiche vengono impostate individualmente (per ciascun ticket) quando un componente dello staff risponde a un ticket o quando il personale dello staff invia un ticket tramite l'interfaccia di amministrazione.

Rimuovi la risposta con citazione

Questa funzione rimuove dalle risposte ai ticket la parte scritta dal cliente nel messaggio originale (= citazione, quote). Questo avviene nel caso di tutti i messaggi di email provenienti da qualsiasi cliente, prima che le risposte siano convertite in ticket usando l'Email Piping o il POP3 Fetching.

Tutte le email che vengono inviate ai clienti iniziano con questa riga:

------ Rispondi sopra questa riga ------

Qualsiasi testo scritto sotto tale riga viene rimosso quando il messaggio di email viene inserito nella risposta al ticket.

Contenuto obbligatorio

Se questa opzione è abilitata, le email che vengono generate tramite Email Piping o POP3 Fetching vengono ignorate se sono prive di contenuto.

Se l'opzione è disabilitata, saranno invece convertite in ticket anche le email prive di contenuto.

File incorporati

Abilitare questa impostazione se si vuole che Hesk salvi i file incorporati nelle email (per esempio delle immagini incorporate) come allegati quando i ticket vengono creati tramite Email Piping o POP3 Fetching.

I file che superano il numero massimo consentito di allegati vengono ignorati.

Attenzione: con Hesk i file allegati hanno priorità sui file incorporati. Per esempio se il limite consentito è di 2 file allegati, quando si riceve una email con 1 file allegato e 3 immagini incorporate, Hesk salverà il file allegato e solo la prima immagine incorporata, le restanti immagini verranno ignorate.

Indirizzi di email multipli

Quando un cliente invia un ticket inserendo nel campo Email più indirizzi di email separati da virgole Hesk, in maniera predefinita, utilizza solo il primo indirizzo e ignora tutti gli altri.

Per consentire ai clienti di inserire indirizzi multipli di email, bisogna abilitare questa opzione mettendo il segno di spunta nella relativa casella.

Conferma indirizzo di email

Se questa opzione è selezionata, il modulo "Invia un ticket" conterrà un campo addizionale denominato "Conferma indirizzo email". Ciò costringe il cliente a digitare due volte il proprio indirizzo di email. In tal modo si riduce la probabilità che vengano commessi errori di digitazione quando si riempie tale campo.
